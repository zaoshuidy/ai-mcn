# 角色与责任矩阵（Role Matrix）

采用 RACI 结构（R=Responsible 执行，A=Accountable 负责，C=Consulted 咨询，I=Informed 知会）。

## 1. 角色定义

### 1.1 项目总控

负责：

- 方案决策；
- 三方向比较与取舍；
- Skill 和工具选型终审；
- 风险判断；
- 验收评分；
- 决定是否进入下一阶段。

### 1.2 人工执行者

负责：

- 小红书真实搜索；
- 链接和截图验证；
- 达人真实性确认；
- 业务判断；
- 平台权限管理；
- 飞书与 GitHub 账号操作；
- 最终人工审核。

### 1.3 技术执行 AI

负责：

- 目录和代码搭建；
- GitHub 工具调研；
- Skill 调研；
- Prompt 和工作流实现；
- 自动化测试；
- 文档生成；
- 日志记录；
- 按要求返工。

技术执行 AI **无权**：

- 擅自改变项目范围；
- 虚构数据；
- 跳过验收；
- 引入未审核第三方组件；
- 直接宣布阶段通过。

## 2. RACI 矩阵

| 活动 | 项目总控 | 人工执行者 | 技术执行 AI |
| --- | --- | --- | --- |
| 项目范围与阶段门禁决策 | A | C | I |
| 三方向方案比较与最终选型 | A | C | R（提供调研与评分） |
| 第三方组件准入终审 | A | C | R（登记、评分卡初评） |
| 仓库目录、代码、文档搭建 | I | I | R |
| 小红书真实搜索与达人真实性确认 | I | R/A | C（提供搜索策略） |
| 链接、截图等证据采集与核验 | I | R/A | C（整理归档） |
| Prompt 与工作流实现 | C | I | R |
| 自动化测试与验证脚本运行 | I | I | R |
| 平台账号、权限、飞书/GitHub 操作 | I | R/A | I |
| 合规风险判断 | A | R | C（提供规则检查） |
| 阶段验收评分与放行 | A | C | I（提供证据，不参与评分） |
| 执行日志与决策日志维护 | I | C | R |
| 最终交付物人工审核 | A | R | I |

## 3. 协作规则

1. 技术执行 AI 的任何产出默认状态为"待验收"，不得自行标记为"通过"；
2. 涉及真实性、合规、密钥的问题，技术执行 AI 必须主动上报，不得默认放行；
3. 项目总控未评分前，不启动下一阶段开发；
4. 人工执行者提供的证据（截图、链接）由技术执行 AI 归档至 `screenshots/` 或对应目录并记录路径。
