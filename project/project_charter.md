# 项目章程（Project Charter）

- 项目名称：mcn-script-assistant
- 项目定位：面向 MCN 商单策划的小红书达人调研、风格分析、脚本生成、合规质检与飞书交付 Agent。
- 当前阶段：Stage 0 - 项目控制台与组件准入机制

## 1. 项目目标

解决 MCN 在食品等品牌商单策划中的核心业务问题：

1. **达人选择无证据**：达人筛选依赖个人经验，缺乏可复核的数据证据链，无法向品牌方解释选择理由；
2. **脚本产能不稳定**：脚本质量依赖个别策划的经验，风格难以复刻，无法规模化生产；
3. **合规风险高**：食品广告受《广告法》等法规约束，人工检查易遗漏违禁词与夸大宣传问题；
4. **交付过程无沉淀**：策划过程分散，无法沉淀为组织可复用的 Skill、Prompt 与流程资产。

项目最终交付一个"Brief 输入 → 达人调研 → 脚本生成 → 合规质检 → 飞书交付"的可复用 Agent 工作流及完整 GitHub 仓库。

## 2. 项目范围

最终项目覆盖以下能力（按阶段逐步实现）：

- Brief 结构化：将品牌方口头/文档 Brief 转为结构化输入；
- 达人调研：制定搜索策略并采集候选达人；
- 达人筛选：对达人进行评分与筛选；
- 风格拆解：拆解达人内容风格，形成风格画像；
- 脚本生成：生成商单视频脚本；
- Humanizer：对文案进行中文人味处理，去除 AI 痕迹；
- 分镜：生成视频分镜与拍摄可行性设计；
- 合规质检：食品广告合规与事实质检；
- Agent 编排：搭建端到端 Agent 工作流；
- 飞书自动写入：将交付物自动写入飞书文档；
- Skill 沉淀：将流程沉淀为通用 MCN Skill；
- GitHub 交付：完整仓库与最终报告。

## 3. 非本阶段范围

Stage 0 **不实现**以下功能，任何提前开发均视为越界：

- 小红书自动爬虫；
- 视频下载；
- 最终脚本生成；
- 飞书 API 调用；
- 完整前端；
- 自动发布内容。

## 4. 核心原则

1. **真实数据优先**：达人、笔记、互动数据必须来自真实渠道，保留链接与截图证据；
2. **人机协作**：AI 负责生成与整理，人负责真实性确认、业务判断与最终审核；
3. **结构化输入输出**：各阶段以 YAML/JSON/CSV 等结构化格式交接；
4. **证据驱动**：所有结论必须可追溯到文件、日志、命令输出或截图；
5. **90 分阶段门禁**：未达门禁不得进入下一阶段；
6. **第三方组件最小权限**：逐项声明并审查网络、Shell、环境变量、文件访问权限；
7. **可替换架构**：第三方组件通过统一适配器接入，任一组件可被替换；
8. **不将模型输出直接视为事实**：模型生成的事实性内容必须经核验后方可作为交付依据。

## 5. 成功标准

项目最终成功需同时满足：

1. **端到端可运行**：从一份真实品牌 Brief 出发，完整走通达人调研到飞书交付全流程；
2. **证据完整**：10 位达人的筛选结论均有真实链接、截图与语料支撑；
3. **质量达标**：生成脚本经 Humanizer 与合规质检后达到可提案水平，通过项目总控评审；
4. **可复用**：形成通用 MCN Skill 与适配器架构，换一个品牌 Brief 可复用流程；
5. **可交付**：GitHub 仓库结构完整、文档清晰、测试与验证脚本真实运行通过，陌生人可按 README 复现。
