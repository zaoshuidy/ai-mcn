# 中文短视频脚本自然化规则（R1 ~ R8，自研）

适用对象：小红书商单口播稿与字幕稿。规则 R1 ~ R6 处理表达层，R7 ~ R8 是
不可违反的红线。每条规则给出问题表现、处理方法与自研示例（示例品牌事实均
取自 data/raw/qingxing_brief.md）。

---

## R1 公文腔清零

- 问题表现：书面连接词与公文动词混入口播——"综上所述""值得注意的是"
  "首先/其次/最后""赋能""助力""打造""致力于""见证"。
- 处理方法：删除或替换为口语过渡（"所以""然后呢""说白了"）；找不到口语
  对应物的，整句删掉通常更自然。
- 示例：
  - before：`综上所述，轻醒酸奶是打造精致生活的完美选择。`
  - after：`所以我现在包里天天装着一杯。`

## R2 排比拆解

- 问题表现：连续 3 个及以上结构一致的分句（"它高蛋白，它0蔗糖，它低负担"
  "适合早餐，适合运动后，适合下午茶"），AI 稿的标志性节奏。
- 处理方法：最多保留两个并列；其余拆成不同句式，或合并成带具体细节的一句。
- 示例：
  - before：`它适合早餐，它适合运动后，它适合下午茶。`
  - after：`早上赶时间、下午犯困、或者练完瑜伽，来一杯都挺合适。`

## R3 空洞修饰删除

- 问题表现：无信息量的形容词/副词堆砌——"非常""超级""极其""完美""惊艳"
  "宝藏""品质卓越""活力满满"。删掉后句意不变的，一律删。
- 处理方法：能删则删；确实需要评价的，换成具体可感的描述（口感、场景、动作）。
- 示例：
  - before：`轻醒0蔗糖希腊酸奶真的非常惊艳。`
  - after：`喝下来觉得口感还挺醇厚。`

## R4 自然停顿补入

- 问题表现：全稿无任何停顿、插入语或话题跳转，一口气念完，像念新闻稿。
- 处理方法：每 2 ~ 3 句允许一个自然停顿点——语气词、插入语或话题标记
  （"对了""说实话""然后""你们懂那种感觉吗"）。停顿必须服务内容，不得为加而加。
- 示例：
  - before：`它有原味、蓝莓、黄桃三种口味，品质非常优秀。`
  - after：`对了，它有原味、蓝莓、黄桃三种口味，我自己偏爱黄桃。`

## R5 口播句长控制

- 问题表现：单句过长（超过一口气能说出的长度），或字幕单条信息过载。
- 处理方法：口播单句建议不超过 25 字，超长拆短；字幕单条建议不超过 20 字，
  一条字幕只承载一个信息点。
- 示例：
  - before：`轻醒酸奶有原味、蓝莓、黄桃三种口味而且0蔗糖高蛋白特别适合早上来不及吃早饭的时候带一杯。`
  - after：`它有原味、蓝莓、黄桃三种口味。0蔗糖、高蛋白，早上来不及吃饭就带一杯。`

## R6 句式节奏打散

- 问题表现：每句结构一致（都是主谓宾起手、长度相近），读起来像列表朗读。
- 处理方法：长短句交错；允许用省略句、设问句、倒装口语句；相邻两句不重复
  相同的主语起手。
- 示例：
  - before：`我今天喝了轻醒酸奶。我觉得口感很好。我推荐大家购买。`
  - after：`今天喝的是轻醒。口感嘛，比我预想的醇厚。想试的可以买黄桃味。`

## R7 事实锚点锁定（红线）

- 规则：原稿中的数字、百分比、品牌名、产品名、规格、口味名、人群标签、
  品牌限定词（如"0蔗糖"）必须**逐字保留**，不得改写、四舍五入、扩写或升级。
- 禁止操作示例：
  - `0蔗糖` → `无糖`（品牌限定词篡改，且触碰 0蔗糖≠无糖 红线）；
  - `22-35岁` → `25-35岁` / `二十多岁`（数字漂移）；
  - `轻醒` → `轻醒优选` / 直接省略品牌名（品牌锚点丢失）；
  - `原味、蓝莓、黄桃三种口味` → `多种口味`（事实粒度丢失）。
- 执行要求：改写前先列出事实锚点清单，改写后逐项核对；清单原样写入输出的
  `preserved_facts` 字段。任何拿捏不准的事实性改写，写入 `possible_fact_drift`
  并降低 `style_match_score`（见"评分约束"）。

## R8 功效零新增（红线）

- 规则：自然化只动表达，不动事实。不得新增原稿没有的功效、对比、数据或
  使用承诺；不得把主观体验升级为客观功效。
- 合规底线（与 config/brief_rules.yaml 一致）：
  - 禁止减肥、燃脂、掉秤、降糖、医疗效果、绝对化表达；
  - `0蔗糖` 不得写成 `无糖`；
  - `饱腹感` 只能停留在主观体验（"我喝完到中午不饿"），不得写成客观功效
    （"抑制食欲""帮助减重"）；
  - 功效表述需有 ProductEvidence 支撑，无证据不得生成。
- 示例：
  - before（原稿）：`它是高蛋白的。`
  - 违规改写：`它能帮你控制体重。`（新增功效，直接判失败）

---

## 编排位置（强制）

本 Skill 是流程**中间节点，永远不是最终节点**。自然化完成后，必须以
`humanized_script` 为对象重新执行三项复检，全部通过才允许进入分镜与交付：

```
xhs-script-humanizer（stage_8）
  → fact_regression（事实回归复检）
  → compliance_review（合规复检）
  → style_consistency_review（风格一致性复检）
```

三项复检不得沿用自然化之前的初检结论；`possible_fact_drift` 非空的输出必须
经人工确认后放行。

## 评分约束

- `possible_fact_drift` 非空时，`style_match_score` 不得大于 0.9；
- 违反 R7 / R8 任一红线的输出为失败品，不得进入复检，直接退回重做。

## 事实漂移上报规范

`possible_fact_drift` 的每条记录写清：原稿表述 → 改写后表述 → 风险点。
示例：`原稿卖点"低负担"改写为"怕负担重"的主观感受表述，未升级为产品承诺，
建议人工复核语境`。
