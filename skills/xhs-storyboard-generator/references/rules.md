# 分镜生成规则（rules）

本规则面向真实小红书拍摄：**手机可拍、1 人可执行、家庭 / 通勤 / 办公室真实场景、道具数量受控、竖屏 9:16、成片总时长与脚本一致**。任何镜头设计如果超出这个执行边界，必须改写，不得带伤交付。

## 1. 可执行性铁律

1. **设备**：仅手机（可加手机三脚架 / 八爪鱼 / 手机稳定器）。不得假设微单、灯光组、录音笔之外的任何专业设备。
2. **人力**：1 人完成出镜 + 拍摄 + 场务。凡是需要第二人操作的镜头（跟拍者、打光师）一律改写为固定机位或第一人称视角。
3. **场景**：只用真实可进入的场景——家中厨房 / 餐桌、通勤路上、电梯、办公室工位、健身房（须确认场地许可）。不得设计影棚、天台、包场餐厅等不可得场景。
4. **道具**：单镜道具 ≤8 件；全片道具须为家庭 / 办公室常备物，出现「轻醒酸奶」以外的品牌实物须做遮挡或去标处理。

## 2. 运镜白名单与黑名单

**默认从白名单选运镜，不得默认电影化运镜。**

白名单（单人手机可完成）：

- 固定（三脚架 / 台面靠置）
- 手持微晃（生活流质感，字幕沉浸式常用）
- 缓慢推进 / 缓慢拉远（手持或支架小幅度）
- 跟随平移（第一人称手持随动作移动）
- 俯拍下移（从俯拍缓缓靠近食物）
- 抬起转向（从手部动作抬到人脸 / 窗外）

黑名单（命中即判失败，见 `validate_output.py` 的 `CAMERA_MOTION_BLACKLIST`）：

| 黑名单词 | 原因 |
| --- | --- |
| 无人机 / 航拍 / 穿越机 | 需要飞行器与飞手，且城市场景多有限飞 |
| 轨道 / 滑轨 | 需要轨道的铺设与第二人操作 |
| 摇臂 / 吊臂 / 升降炮 | 剧组级设备 |
| 斯坦尼康 | 需要专业背心稳定器与操作员 |
| 机械臂 | 影棚设备 |
| 索道 / 飞猫 | 大型外景设备 |

## 3. 镜头时长与数量

镜头平均时长以真实达人证据为准（来源：`data/processed/stage_3_top3_video_timelines.json`）：

| 达人 | 形态 | 平均镜头时长 | 成片时长 | 镜头数（segments） |
| --- | --- | --- | --- | --- |
| 欧盈Kelly | 口播型（voiceover） | 2.4s | 153.5s | 10（叙事段） |
| 小季没烦恼 | 字幕型（subtitle_immersive） | 2.5s | 114.0s | 8（叙事段） |
| 一只牛 | 字幕型（subtitle_immersive） | 2.0s | 54.7s | 7（叙事段） |

- 镜头数估算：`镜头数 ≈ target_duration / style_profile.avg_shot_s`，按 beat 边界取整。
- 字幕沉浸式（无口播）：单镜 1.5–3s，靠硬切 / 跳切保持节奏。
- 口播型：单镜 2–4s，口播句与镜头切换对齐，避免一句话切超过 2 次。
- 校验器对全片平均镜头时长给出 1.0–4.0s 的提示区间（warning，不阻断）。

## 4. 时长与产品露出

1. 分镜时间轴必须从 0.0 开始，镜头间**连续、无重叠、无空洞**；单镜 `duration` 必须等于 `end_time - start_time`。
2. `actual_total_duration_s`（= 末镜头 `end_time`）与 `target_duration_s` 的偏差 ≤3s。
3. 产品露出镜头（`product_exposure=true`，或 `product_state` 为 入镜静置 / 手持展示 / 特写展示 / 开封·使用中 / 食用·饮用中）的 `start_time` **不得早于**脚本的 `product_first_appearance_s`。
4. 植入结构参考小季没烦恼的真实范式：「需求场景 → 产品解法」——先用 1 镜建立需求（如「上午想吃又怕太甜」），再给产品特写与卖点字幕；单段植入建议 ≤ 成片时长的 15%。
5. 开场 hook 二选一：成品先行（一只牛范式，0s 给成品特写）或口语化问候 + 标题卡（欧盈Kelly 范式）；禁止空镜超过 3s 才进入正题。

## 5. 合规红线（每镜必查）

背景：轻醒 0 蔗糖高蛋白希腊酸奶。以下规则逐镜作用于 `spoken_line`、`on_screen_text`、`visual` 中的文字元素：

1. **禁止**减肥 / 瘦身 / 减重 / 燃脂 / 掉秤类表述。
2. **禁止**降糖 / 降血糖 / 控制血糖等血糖功效；「控糖」只能作为人群标签，不得进入卖点。
3. **禁止**治疗 / 治愈 / 预防疾病 / 药用 / 疗效等医疗表述。
4. **禁止**绝对化用语：最健康 / 第一 / 100% / 完全无负担 / 绝对。
5. 「0 蔗糖」**不得写作「无糖」**；「高蛋白」「0 蔗糖」等品牌宣称须注明依据来源（ProductEvidence / 品牌方资料），无依据不得扩展数值（蛋白质克数、热量等）。
6. 「饱腹感」「低负担」为受限主观体验：必须使用个人体验语境（「我觉得」「对我来说」），不得延伸为「减重」「抑制食欲」「不长胖」「一整天不饿」。
7. 不得贬低竞品（碾压 / 吊打 / 完胜 / 比其他品牌好）。
8. 每镜 `compliance_note` 写明本镜自查结论；无卖点信息写「无卖点表述，无合规风险」。

## 6. 字幕与口播规范

- 字幕沉浸式：单条字幕 ≤15 字（一只牛真实节奏），只做制作说明与口感点评；`spoken_line` 置 null。
- 口播型：口播为叙事主线，字幕承担时间戳 + 情绪梗（参考欧盈Kelly），字幕不得复述完整口播。
- 结尾可用固定结束语 + 产品收尾展示（一只牛 / 欧盈Kelly 均有此结构），结束语不得抄袭达人原文。

## 7. 风格证据要求

每镜 `style_evidence` 必须可溯源，格式：`<依据内容>（<来源文件>）`。允许的来源：

- `data/processed/stage_3_top3_video_timelines.json`（3 条真实视频时间线）
- `data/processed/stage_3_deep_review_15.json`（15 位达人深核）
- `style_profile` 输入中由上游 Skill 给出的字段

禁止编造达人数据、时长或结构结论。
