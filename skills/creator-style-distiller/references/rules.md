# 蒸馏规则（rules）

本文件是 creator-style-distiller 的取证与表达规范。任何一条规则都不允许被"效率"绕过。

## 一、五条禁令（红线，违反即判输出无效）

1. **禁逐句复制**：不得把达人口播台词、屏幕字幕的原句搬进输出。所有表达必须转述。
   工程底线：输出 JSON 中任何字符串与输入 transcript/on_screen_text 不得出现
   **超过 15 字的连续相同子串**（`scripts/validate_output.py --input` 强制校验）。
2. **禁固定口头禅复制**：达人的标志性口头禅、固定结束语、个人梗（如问候黑话、人格标签自嘲）
   属于身份资产，只能写入 `creator_specific_elements_not_to_copy`，不得进入
   `reusable_style_rules`，更不得出现在给轻醒的脚本建议里。
3. **禁身份仿冒**：输出是"风格分析文档"，不是达人稿件。不得模仿达人第一人称身份写成品文案，
   不得生成"以该达人口吻可直接发布"的脚本。
4. **禁声称达人本人创作**：交付物及其元信息不得暗示内容由达人本人产出、背书或授权。
5. **禁无证据的风格推断**：每条规则必须能回溯到输入证据（timeline 段、时间戳、字段）。
   没有证据支撑的"风格判断"（包括从粉丝量、人设标签脑补出来的特征）一律不得写入；
   证据不足时降低 `confidence` 并在对应字段注明证据缺口。

## 二、取证规则

- **证据优先级**：timeline 分段时间戳 > keyframe 统计 > transcript summary > 画像文字。
  冲突时以带时间戳的证据为准。
- **ASR 可靠性分层**：timeline 标注 `asr_reliability` 为 unreliable 的转写内容（如 BGM 幻觉文本）
  不得作为口播证据；只能依据 `transcript_summary` 与 `on_screen_text` 的已核验部分。
- **≥3 条交叉验证**：写入 `reusable_style_rules` 的模式应在 ≥3 条视频中重复出现；
  仅单条出现的写法可进对应 patterns 字段并标注样本量，confidence 相应下调。
- **时间戳必填**：`evidence_timestamps` 汇总输出所依据的关键秒点（单点或 [起, 止] 区间），
  非空；每个 pattern 尽量附自己的 `timestamps`。
- **商业信号如实**：`commercial_integration_patterns` 只描述证据里存在的商业表达
  （含"自用展示、是否软广无法确认"这类不确定状态），不得替达人断定有/无商单。

## 三、转述规范

- 用"手法语言"替代"台词语言"：描述结构与功能（例："以时间戳字幕推进一日叙事"），
  不引用原句。示意见 `references/examples.md`。
- 引用证据用定位而非原文：写"第 5 段（38-48s）早餐场景口播提及固定搭配"，
  而不是抄下那句口播。
- 字段值一律中文；度量单位用秒（s）；时间戳为视频内秒数（浮点）。

## 四、轻醒品牌侧约束（风格建议落点时遵守）

- 场景锚点：早餐 / 一人食 / 通勤 / 办公室下午茶 / 运动后加餐。
- 合规红线：禁止减肥、燃脂、掉秤、降糖、医疗效果、绝对化表达；
  "0蔗糖"不得写成"无糖"；任何功效表述需 ProductEvidence 支撑。
- 风格蒸馏本身不产出功效话术，但 `reusable_style_rules` 若指向植入位置建议，
  须与上述红线兼容（例如只建议"生活流自然位置植入"，不建议"功效对比植入"）。

## 五、置信度规则

- 0.8-1.0：≥3 条视频一致证据，且关键字段有双时间戳佐证。
- 0.5-0.8：2 条一致或单条多段佐证。
- <0.5：单点证据 / ASR 不可靠 / 推断成分高——此时必须在输出中写明证据缺口。
